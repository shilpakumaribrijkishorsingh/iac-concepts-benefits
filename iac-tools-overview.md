\# Overview of Infrastructure as Code (IaC) Tools



Infrastructure as Code (IaC) is the practice of managing and provisioning infrastructure through code instead of manual processes. It helps teams automate deployments, maintain consistency, and reduce human errors. This document summarizes the most commonly used IaC tools and their roles.



\## Terraform



\### Features

\- Open source tool developed by HashiCorp

\- Uses a declarative language called HCL (HashiCorp Configuration Language)

\- Supports multiple cloud providers like AWS, Azure, and GCP

\- Maintains infrastructure state using state files

\- Supports reusable modules for standardized infrastructure



\### Use Cases

\- Provisioning infrastructure across multiple cloud platforms

\- Creating consistent environments such as development, testing, and production

\- Integrating infrastructure provisioning into CI/CD pipelines



\### Advantages

\- Cloud agnostic and vendor neutral

\- Strong community and provider support

\- Ability to preview changes before applying them

\- Easy to manage infrastructure as version controlled code
